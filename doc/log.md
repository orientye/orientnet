# 设计目标
- level: 支持
- 目的地:
    - local file(network, db无意义)
- 滚动方式:
    - 按时间大小滚动
    - 按大小滚动
- thread-safe: 支持
- crash-safe: 暂不支持
    - 关键日志额外立即存储?
    - 调试coredump文件?
- 同步与异步
    - 支持同步
    - 支持异步
- 格式
    - 行
    - 时间(时区GMT?) 线程ID

- 主要思想
    - 一个进程里, 多个生产者(即多个线程), 一个消费者(log线程)
        - 一个消费者线程能否对应多个目的文件? 或者说每个进程能写多个日志文件吗?
    - 多缓冲
    - 有序
